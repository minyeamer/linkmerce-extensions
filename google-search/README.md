# 구글 검색 자동화

> 여러 키워드를 순차 검색하여 Google 검색 결과를 **CSV 또는 JSON 파일**로 자동 수집하는 크롬 확장 프로그램

## 주요 기능

- 키워드를 여러 개 입력하면 순서대로 검색하여 하나의 파일로 합산 저장
- 다음 페이지가 없어질 때까지 자동으로 이어서 수집하는 **전부 수집 모드**
- 수집 중 CAPTCHA가 감지되면 최대 2분간 대기 후 자동 재개
- 수집 중단 후 중단된 지점부터 **이어서 수집** 가능
- 수집 완료 결과를 내부 저장소에 보관 — 자동 다운로드 실패 시 수동 재다운로드 가능
- 페이지 이동 시 지정된 최소·최대 범위 내 **랜덤 딜레이**를 적용하여 과부하 방지

## 아키텍처

```
[팝업 UI (popup.html / popup.js)]
      │  chrome.runtime.sendMessage(startCollection)
      ▼
[Service Worker (background.js)]
      │  전용 수집 탭(active:false) 생성 및 관리
      │  탭 내 content_script.js를 통해 결과 추출
      │   ├─ parseCurrentGooglePage()  검색 결과 파싱
      │   ├─ detectCaptchaOrVerification()  CAPTCHA 감지
      │   └─ findNextPageUrl()  다음 페이지 링크 탐색
      │  결과 → chrome.storage.local 저장
      ▼
[chrome.downloads API]  CSV / JSON 파일로 저장
```

## 수집 데이터

| 컬럼 | 설명 |
|------|------|
| `query` | 키워드 |
| `page` | 페이지 번호 |
| `rank` | 해당 페이지 내 순위 |
| `title` | 검색 결과 제목 |
| `url` | 실제 URL |
| `display_url` | 표시 URL (cite 요소) |
| `snippet` | 본문 요약 |
| `collected_at` | 수집 일시 (ISO 8601) |

## 설치 (개발 버전)

1. `chrome://extensions` 접속
2. 우측 상단 **개발자 모드** ON
3. **압축해제된 확장 프로그램을 로드합니다** 클릭
4. `google-search/` 폴더 선택

## 사용 방법

1. 확장 프로그램 아이콘을 클릭하여 팝업을 엽니다.
2. **키워드 목록**에 키워드를 한 줄에 하나씩 입력합니다.
3. **수집 설정**을 조정합니다.
   - 최소·최대 대기(ms): 페이지 이동 전마다 적용할 랜덤 딜레이 범위
   - 최대 페이지 수: 전부 수집 모드가 꺼진 경우 한 키워드당 검색할 최대 페이지 수
   - 전부 수집: 다음 페이지 링크가 있는 동안 계속 수집
4. **내보내기 설정**에서 저장 파일명과 형식(CSV 또는 JSON)을 선택합니다.
5. **수집 시작**을 누릅니다. 수집하는 동안 별도 탭이 백그라운드에서 작동합니다.
6. 수집이 완료되면 파일이 Chrome 기본 다운로드 폴더에 저장됩니다.

### CAPTCHA 대응

- 수집 중 CAPTCHA가 감지되면 팝업에 안내가 표시되며, 최대 **2분간** 대기합니다.
- 이 시간 내에 수집 탭에서 직접 인증을 완료하면 자동으로 재개됩니다.
- 2분이 지나도 해제되지 않으면 해당 키워드 검색을 중단합니다.

### 이어서 수집

- **수집 중지**를 누르면 현재 페이지 완료 후 중단되며, 중단 지점이 저장됩니다.
- 다시 팝업을 열면 **이어서 수집** 버튼이 나타나고, 중단된 키워드·페이지부터 재개합니다.
- 이미 수집된 행은 이어서 수집 시 결과에 포함됩니다.

### 결과 다시 받기

- 자동 다운로드가 실패하더라도 최근 수집 결과는 확장 내부 저장소에 보관됩니다.
- 팝업의 **최근 결과 CSV** 또는 **최근 결과 JSON** 버튼으로 수동 재다운로드가 가능합니다.

### 진행 상태 초기화

- 수집이 멈춰서 응답이 없을 때 **⟳ 진행 상태 초기화** 버튼으로 모든 상태를 리셋합니다.
- 초기화 후 처음부터 새로 수집을 시작할 수 있습니다.

## 다운로드 위치

- Chrome의 **다운로드 설정**에 따라 저장 위치가 결정됩니다.
- 기본적으로 Windows에서는 `C:\Users\<사용자명>\Downloads` 폴더에 저장됩니다.
- Chrome 설정에서 **다운로드 전 저장 위치 확인** 옵션이 켜져 있으면 저장 대화상자가 표시됩니다.

## Excel 변환

CSV 파일은 Excel에서 바로 열 수 있습니다. `.xlsx`로 변환하려면 아래 Python 스크립트를 사용합니다.

```python
import pandas as pd

df = pd.read_csv("검색결과.csv")
df.to_excel("검색결과.xlsx", index=False)
```


## 이번 버전에서 추가된 점

- 키워드 여러 개 입력 가능
- 키워드는 파일명 대신 **결과 데이터의 `query` 열**에 저장
- 기본 파일명은 `검색결과.csv` 또는 `검색결과.json`
- `다음` 링크가 없어질 때까지 계속 가는 **전부 수집 모드** 지원
- 팝업 UI 개선 및 설정 저장 기능 추가
- 최근 수집 결과를 확장 내부에 기억하고, 다운로드 실패 시 다시 받기 가능
- 페이지 간 딜레이를 최소/최대 ms 범위의 랜덤 값으로 적용

## 설치

1. 크롬에서 `chrome://extensions`를 엽니다.
2. 우측 상단의 **개발자 모드**를 켭니다.
3. **압축해제된 확장 프로그램을 로드합니다**를 클릭합니다.
4. `c:\Users\user\code\VSL\google-search` 폴더를 선택합니다.

## 사용 방법

1. 확장 프로그램 아이콘을 클릭합니다.
2. 키워드를 한 줄에 하나씩 입력합니다.
	- 예: `VSL "20년"`
	- 예: `VSL "20여년"`
3. 페이지 간 대기 시간, 최대 페이지 수, 전부 수집 여부를 설정합니다.
	- 페이지 간 대기는 고정값이 아니라 **최소~최대 범위 랜덤값**입니다.
4. 저장 파일명을 입력합니다. 기본값은 `검색결과`입니다.
5. 형식은 기본적으로 `CSV`를 추천합니다.
6. `수집 시작`을 누르면 키워드별로 Google 검색 탭을 열고 결과를 하나의 파일로 저장합니다.
7. 자동 저장이 실패했더라도 팝업의 **최근 결과 CSV/JSON** 버튼으로 다시 내려받을 수 있습니다.

## 다운로드 위치

- 이 확장 프로그램은 Chrome의 다운로드 기능을 사용합니다.
- 따라서 저장 위치는 **Chrome 다운로드 설정**을 따릅니다.
- 별도 변경이 없다면 보통 Windows에서는 `C:\Users\user\Downloads` 아래로 내려갑니다.
- 다만 Chrome에서 **다운로드 전에 저장 위치 확인** 옵션이 켜져 있으면, 저장 대화상자가 뜨고 사용자가 직접 저장을 눌러야 실제 파일이 생깁니다.

팝업에는 이제 아래 정보가 표시됩니다.

- 저장 요청한 파일명
- 다운로드 상태(`complete`, `interrupted` 등)
- 브라우저가 실제로 기록한 저장 파일명

즉, 앞으로는 `검색결과.csv`가 실제로 저장됐는지 팝업에서 더 분명하게 확인할 수 있습니다.

## CAPTCHA / 확인 페이지 대응

- 검색 중 CAPTCHA 또는 확인 페이지가 감지되면 확장 프로그램은 **2분 동안 대기**합니다.
- 이 시간 동안 사용자가 직접 인증을 완료하면, 이후 같은 페이지에서 다시 결과 추출을 시도합니다.
- 2분 후에도 CAPTCHA가 계속 보이면 해당 키워드 검색은 중단됩니다.

## 파일 구조

CSV/JSON에는 아래 정보가 들어갑니다.

- `query`
- `page`
- `rank`
- `title`
- `url`
- `display_url`
- `snippet`
- `collected_at`

즉, 파일명에 키워드를 넣지 않아도 나중에 어떤 행이 어떤 키워드 결과인지 구분할 수 있습니다.

## 최근 결과 기억 기능

- 수집이 끝나면 가장 최근 결과 행 목록과 메타데이터를 확장 내부 저장소에 보관합니다.
- 브라우저 다운로드가 실패하거나 저장 대화상자를 닫았더라도, 다시 팝업을 열어 최근 결과를 수동으로 다시 받을 수 있습니다.
- 이 저장소는 임시 작업용이며, 아주 큰 데이터셋을 장기간 보관하는 용도는 아닙니다.

## Excel 저장

브라우저 확장에서 직접 `.xlsx`를 만드는 대신, 우선 CSV를 안정적으로 저장하도록 했습니다. CSV는 Excel에서 바로 열 수 있습니다.

원하면 아래 보조 스크립트로 `.xlsx` 변환도 가능합니다.

- `csv_to_excel.py`

예:

```python
import pandas as pd

df = pd.read_csv("검색결과.csv")
df.to_excel("검색결과.xlsx", index=False)
```

## 전부 가져오기 모드에 대한 설명

`전부 수집`을 켜면 확장 프로그램은 다음 페이지 링크가 계속 존재하는 동안 순차적으로 이동합니다.

다만 아래 경우에는 "전부"가 보장되지 않습니다.

- Google이 중간에 다른 레이아웃을 보여주는 경우
- 페이지 구조가 바뀐 경우
- 검색 결과가 동적으로 달라지는 경우
- 자동화 제한, 확인 페이지, 비정상 응답이 나타나는 경우

그래서 구현은 **가능한 한 끝까지 시도**하는 방식이고, 절대적으로 무한/완전 수집을 보장하는 것은 아닙니다.

## 한계

- 광고, 쇼핑, 지식패널 등 특수 결과 블록은 기본적으로 제외될 수 있습니다.
- Google DOM 구조가 바뀌면 선택자 수정이 필요할 수 있습니다.
- 프록시/탐지 우회/로그인 쿠키를 이용한 자동화는 포함하지 않습니다.
